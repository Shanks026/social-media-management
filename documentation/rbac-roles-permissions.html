Tercero · Access Control Reference

System Roles & Permissions

Who can do what inside a single agency workspace. The system role decides access; the functional role is a cosmetic job title and never affects permissions.

Owner — runs the company Admin — runs the work Member — does assigned work

Owner

The founder/co-founder who signed up. Exactly one. Owns billing, the team, and the workspace itself.

Admin

A trusted operational deputy. Full command of client work — but nothing about the company.

Member

Delivery staff. The default for everyone who joins. Drafts work; needs approval to send it out.

01

The three system roles

Highlights of what each tier can and cannot do.

Owner

Founder / co-founder · account holder

Exactly 1 · immutable
  • ✓Everything an Admin can do
  • ✓Onboard team & promote/demote roles
  • ✓Billing — the Tercero subscription
  • ✓Edit workspace settings & branding
  • ✓Configure the invoice signatory
  • ✓Delete the workspace

Admin

Operational deputy · granted by promotion

Many · owner-promoted
  • ✓Full CRUD on clients & deliverables
  • ✓Full finance + issue invoices
  • ✓All documents, incl. confidential
  • ✓Proposals, prospects, reports, campaigns
  • ✕Team management
  • ✕Billing · workspace edit · signatory · delete

Member

Delivery staff · default on join

Many · default role
  • ✓View all clients
  • ✓Draft & edit deliverables (needs approval)
  • ✓Work within campaigns · notes · meetings
  • ✓Documents — by per-member level
  • ✕Finance, proposals, prospects, reports
  • ✕Confidential docs · create/delete clients

03

Functional roles

Cosmetic job titles — a badge only, never an access control.

These do not affect permissions. A Member who is a "Finance Manager" still has no finance access — access comes only from the system role above. The list is a curated default; an owner can type a Custom… title for any agency type.
